### 기본 사항

---

```
---
여기에 머리말을 yaml 형태로 작성
서두에 --- 두 줄을 입력하면 빌드 대상이 됨
---
아래에 markdown/html 작성. Liquid 태그 사용 가능
```

- assets 폴더 안에 있는 파일은 서두에 --- 두 줄이 없어도 배포 가능
- assets 폴더 밖에 있는 파일은 서두에 --- 두 줄이 있어야 배포 가능
- \_layouts 폴더에는 layout들을 넣음
  - 본문이 들어갈 자리에 `{% raw %}{{content}}{% endraw %}` 작성
  - layout 사용은 페이지의 머리말에 layout 설정 예) `layout: post.html`
- \_includes 폴더 안에는 자주 사용할 템플릿을 넣음
  - 템플릿 사용은 `{% raw %}{% include 파일 경로 %}{% endraw %}`
  - 예) \_includes/footer.html을 사용하려면 페이지에서 넣을 부분에 `{% raw %}{% include footer.html %}{% endraw %}`
- \_posts 폴더는 글 작성 부분
- \_site 폴더는 빌드 파일들
- 그 외 원하는 폴더 사용 가능 (단, \_로 시작하면 안됨)
- 기본 폴더 구조(**굵은 글씨**는 폴더)
  - **assets**
    - **css**
    - **images**
    - **js**
  - **\_includes**
  - **\_layouts**
  - **\_posts**
  - index.html
  - \_config.yml
  - Gemfile
  - 그 외

### config.yml

---

블로그 예시를 봤는데 category 파트가 있길래 따라 했다

```
# category
categories_main: [에러, 공부]
categories_error: [SpringBoot, JSP, ThymeLeaf, Vue, React, SQL, 그 외]
categories_study: [Jekyll, 알고리즘, 예상질문]
```

카테고리 구분에 사용할 것 같다

### \_layouts/default.html

---

\_layouts 폴더가 없어서 일단 만들어줬다

그리고 default.html 파일 생성

내용은 일단 그대로 가져왔다..ㅎ

```
<title>{% if page.title %}{{page.title}} | {% endif %} {{site.title}}</title>
```

page.변수명은 최종 페이지의 변수

이건 --- 두 줄 안에서 선언하는 변수인 것 같다

site.변수명은 \_config.yml에서 정의한 변수

css/scss나 js부분은 나중에 하고 틀만 잡아보겠다

### \_layouts/post.html

---

post.html 작성

layout은 default

### \_layouts/page.html, \_layouts/home.html

---

layout은 default

content만 출력

### \_includes/sidebar/sidebar1.html, \_includes/sidebar/sidebar2.html

---

sidebar include를 하는 부분이다

component include는 일단 \_includes 폴더에 html 템플릿을 만들어두고

사용할 레이아웃에서 `{% raw %}{% include 파일명.html %}{% endraw%}` 이렇게 사용하는 것 같다

그런데 이쯤에서 실행해보는데

왜 레이아웃을 못 찾는걸까..
