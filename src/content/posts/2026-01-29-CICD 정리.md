### CI/CD

> DevOps (Developer + Operation)

1. CI
   - 코드가 정상적으로 통합이 되어있는지 확인 => 자동 검증
   - 시점 : git push (commit + push) / merge
     - 코드 체크아웃
     - 빌드
     - 테스트
     - 오류 검증
       - script (deploy.yml) : 순서 : name
       - jenkins : stage
   - **정상적인 코드만 서버로 전송**
2. CD (Delivery) => Deployment
   - 검증된 코드만 서버로 자동 배포
   - 시점 : CI가 완성된 이후
   - JAR(WAR) 전송 / docker 이미지 / 서버 실행 / 재시작
   - 자동화 (운영자 거치지 않고)

|          | CI                                                                   | CD               |
| -------- | -------------------------------------------------------------------- | ---------------- |
| 의미     | 소스 통합                                                            | 배포             |
| 대상     | 코드 품질                                                            | 서비스 운영      |
| 실행시점 | commit/push 등                                                       | CI 성공          |
| 도구     | Jenkins, Github Actions                                              | Jenkins, Argo CD |
| 결과값   | jar/war/image<br><br>jar : ThymeLeaf<br>war : JSP<br>SpringFramework | 실행 중 서비스   |

### 동작 순서 (CI/CD)

개발자
| => 소스 완료 : commit / push
Git Hub
| => webhook 감지 => 알림
Jenkins ------------------------ CI 영역 => 별도의 서버

1. CheckOut => SCM
2. Build => 수정
3. Docker Build => 이미지
4. Docker Hub => push

|
Docker Hub
| => docker pull
EC2 ---------------------------- CD 영역 (war / 이미지파일)
| => docker run / docker compose up
사용자에서 브라우저 사용이 가능

필요한 도구

1. 개발자 => git / Spring-Boot
2. docker / docker-compose / jenkins / minikube

---

### 1. 도구의 역할

1. Git => 저장 / 팀별 공유 == 협업 / 브런치 전략
   - 소스 저장소
   - Jenkins 트리거 역할 (연결 역할)
2. Jenkins
   - CI : 빌드 / 테스트
   - CD :
     - Docker 이미지 생성
     - Docker Hub Push
     - 명령 => 서버 배포 명령 : docker images / docker ps
3. Docker Hub
   - 이미지 저장소 / 버전 관리 / 롤백 가능
     - 같은 이름으로 push => 자동으로 수정
4. EC2 : 실제 서비스 관련 서버, 컨테이너 실행, 사용자 요청 처리
5. Docker 사용 이유
   1. 환경 동일
   2. 이미지 하나로 해결 => 실행 환경 + 소스
6. Docker Compose : 여러 개의 war를 동시에 실행 : MSA
7. Docker
   - App 실행 환경을 이미지로 표준화
   - 실행 환경 패키징
   - OS/APP/JDK 통합
   - 어디서나 동일 실행을 보장 => 컨테이너
8. Docker Compose
   - 컨테이너 : 여러 개를 한 번에 관리하는 실행 스크립트
     - 포트 / 네트워크 설정
     - 실행 순서 관리
     - 재시작 / 종료 자동화
     - Docker 위에서 동작
     - 운영 서버 실행 담당

**CI = 코드 검증 자동화 / CD = 배포 자동화 / Docker = 실행환경 표준화**
