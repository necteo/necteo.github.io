테마 기본 댓글 설정을 살펴보니 Twikoo라는 걸 쓰고있었다

따거들이 주로 쓰는 듯 하다

그래도 Giscus를 넣어보고 싶어서 커스텀 시작

### 과정

`src/config.ts`에서 `commetConfig` 수정

```ts
export const commentConfig: CommentConfig = {
	enable: true, // 启用评论功能。当设置为 false 时，评论组件将不会显示在文章区域。
	// twikoo: {
	// 	envId: "https://twikoo.vercel.app",
	// 	lang: SITE_LANG,
	// },
	giscus: {
		repo: "necteo/necteo.github.io",
        repoId: 'R_kgDOQl-0Gg',
        category: 'Announcements',
        categoryId: 'DIC_kwDOQl-0Gs4Cz2kj',
        mapping: 'pathname',
        strict: '0',
        reactionsEnabled: '1',
        emitMetadata: '0',
        inputPosition: 'bottom',
        theme: 'preferred_color_scheme',
        lang: 'ko',
        crossorigin: "anonymous",
        loading: 'lazy'
	}
};
```

giscus 설정에 필요한 리소스값 저장

`src/components/comment/Twikoo.astro` 파일명을 `Giscus.astro`로 변경

그리고 내용 변경을 해야하는데

자세히는 모르겠지만 twikoo는 초기화 과정을 직접 해야하는 부분이 많은 것 같다고 판단했다

일단 giscus는 jekyll때도 스크립트 하나만 추가해도 작동되는걸 봤기 때문에

긴 초기화 설정 코드를 주석으로 밀어두고 giscus 설정 script를 추가했다

```js
<script src="https://giscus.app/client.js"
        data-repo={config.repo}
        data-repo-id={config.repoId}
        data-category={config.category}
        data-category-id={config.categoryId}
        data-mapping={config.mapping}
        data-strict={config.strict}
        data-reactions-enabled={config.reactionsEnabled}
        data-emit-metadata={config.emitMetadata}
        data-input-position={config.inputPosition}
        data-theme={config.theme}
        data-lang={config.lang}
        crossorigin={config.crossorigin}
        async>
</script>
```

무사히 작동하는 모습이다
