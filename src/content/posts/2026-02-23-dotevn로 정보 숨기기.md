`Spring Boot`나 `node.js`로 서버를 돌리는데

DB의 username과 password를 숨기고 싶었다.

`application.yml`은 솔직히 github에 안올리는 건 어렵다고 생각한다.

은근 복잡한 설정이 많은데 이걸 똑같이 알아서 하라는 건 좀

그래서 알아본 방법

### 방법

#### 1. 시스템 환경변수

OS에서 직접 설정한다.

```bash
# Windows PowerShell
$env:DB_PASSWORD="비밀번호"

# Linux/Mac
export DB_PASSWORD=비밀번호
```

```ts
const db = await mysql.createConnection({
	password: process.env.DB_PASSWORD,
});
```

#### 2. Node.js 실행 시 전달

```bash
DB_PASSWORD=비밀번호 node app.js
```

#### 3. config.json

```json
{
	"db": {
		"host": "localhost",
		"user": "root",
		"password": "비밀번호",
		"database": "mydb"
	}
}
```

```ts
import config from './config.json';

const db = await mysql.createConnection(config.db);
```

`.gitignore`에 `config.json` 추가

#### 4. package.json scripts에서 전달

```json
{
	"scripts": {
		"start": "DB_PASSWORD=비밀번호 node app.js"
	}
}
```

`.gitignore`에 `package.json` 추가

#### 5. dotenv + .env

```bash
npm install dotenv
```

**.env 파일 생성**

```
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=비밀번호
DB_NAME=데이터베이스명
```

```ts
import dotenv from 'dotenv';
import mysql from 'mysql2/promise';

dotenv.config();

const db = await mysql.createConnection({
	host: process.env.DB_HOST,
	user: process.env.DB_USER,
	password: process.env.DB_PASSWORD,
	database: process.env.DB_NAME,
});
```

`.gitignore`에 `.env`추가

### 마무리

솔직히 `config.json` 만들어서 `.gitignore`에 추가하는 거랑 비슷한 거 같은데

json은 뭐 파싱도 해야되고 언어마다 쓰는 법이 다르고 해서

범용성이 좋아서 사용하는 것 같다.
